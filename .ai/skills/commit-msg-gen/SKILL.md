# Commit Message Generator

코드 변경 내역을 바탕으로 사람이 작성한 것보다 더 상세하고 정확한 커밋 메시지 본문을 생성하는 스킬이다.

## 핵심 기능

- **변경 파일 분석**: `git diff`를 통해 수정, 추가, 삭제된 파일을 식별한다.
- **의미적 요약**: 단순 코드 변경이 아닌, 기능적/논리적 변화(메서드 추가, 로직 변경 등)를 추출한다.
- **상세 본문 생성**: 변경된 각 파일에 대해 어떤 부분이 왜 바뀌었는지 상세히 기술한다.

## 사용 방법

1. `git diff --cached` (또는 `git diff`) 명령어로 변경 내역을 확보한다.
2. 이 스킬을 호출하여 변경 내역을 전달한다.
3. 생성된 텍스트를 커밋 메시지 본문에 삽입한다.

## 결과물 형식

```markdown
[Short summary of changes]

### 상세 변경 내역

#### [파일명]
- [수정 사항 1]: 상세 설명
- [수정 사항 2]: 상세 설명

#### [파일명 2]
- [수정 사항 1]: 상세 설명
```
