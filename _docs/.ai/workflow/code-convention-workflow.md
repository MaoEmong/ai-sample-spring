# 코드 컨벤션 워크플로우

## 트리거
- `src/main/java/com/example/demo/**` 내 Java 파일 생성/수정/리뷰 요청

## 단계
1. 변경 대상 도메인 패키지 식별 (`board`, `user`, 신규 도메인)
2. `_docs/.ai/references/code-rule-ai.md` 로드
3. 체크리스트 기준으로 대상 코드 검증
4. 위반 사항을 최소 패치로 수정
5. 최종 응답 전 체크리스트 재검증

## 패치 전략
- 작고 국소적인 수정 우선
- 무관한 리팩터링 금지
- 기존 코드베이스 네이밍/어노테이션 스타일 유지

## 리뷰 전략
- 심각도 기준으로 보고
  - Blocker: 컴파일/런타임/스키마 문제
  - Major: 계층/트랜잭션/DTO 경계 위반
  - Minor: 네이밍/스타일 불일치

## 완료 기준
- 변경 파일에 체크리스트 위반이 남아있지 않다.
- Controller가 Entity를 직접 노출하지 않는다.
- Service 트랜잭션 기본값이 맞다.
- 신규/수정 DTO가 중첩 static class 규칙을 따른다.
